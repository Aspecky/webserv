What is Docker?

Docker is a platform that enables developers to package applications into containers—standardized executable components combining application source code with the operating system libraries and dependencies required to run that code in any environment.

Containers simplify delivery of distributed applications, and have become increasingly popular as organizations shift to cloud-native development and hybrid multicloud environments.

Core Docker Concepts

Images

Read-only templates containing instructions for creating containers. Built from Dockerfiles, they include everything needed to run an application.

Containers

Runnable instances of images. Isolated from other containers and the host machine, they can be started, stopped, moved, and deleted.

Volumes

Persistent data storage mechanism. Volumes exist outside containers and survive container deletion, enabling data persistence.

Networks

Enable communication between containers. Docker provides several network drivers for different use cases.

Docker Compose

Tool for defining and running multi-container applications. Uses YAML files to configure application services.

Dockerfile

Text document containing commands to assemble an image. Each instruction creates a layer in the image.

Key Comparisons

Aspect Option A Option B
Virtualization Virtual Machines - Full OS isolation, heavier Docker - OS-level virtualization, lightweight
Storage Bind Mounts - Direct host filesystem access Docker Volumes - Managed by Docker, portable
Networking Host Network - Uses host's network stack Docker Network - Isolated virtual networks
Secrets Environment Variables - Visible in process list Docker Secrets - Encrypted, secure storage

Inception Project Overview

Version: 5.2

Objective: Build a small infrastructure composed of different services using Docker Compose. The entire project must be done in a virtual machine, following specific Docker best practices.

Docker Docker Compose NGINX WordPress MariaDB TLS

Project Philosophy

This project emphasizes proper Docker practices and system administration skills. It's not just about getting services running—it's about understanding containerization, security, networking, and infrastructure design.

Students are encouraged to use AI tools wisely: for reducing repetitive tasks, learning prompting skills, and understanding how AI systems work. However, all AI-generated content must be fully understood and peer-reviewed.

Mandatory Requirements

Service Architecture

🌐 NGINX

Web server with TLSv1.2/1.3 only. Acts as the single entry point on port 443.

📝 WordPress

Content management system with php-fpm. No nginx inside this container.

🗄️ MariaDB

Database server. Must have two users (one admin, username cannot contain 'admin').

Key Technical Requirements

  • Each service runs in a dedicated container
  • Build from Alpine or Debian (penultimate stable version)
  • Write custom Dockerfiles for each service
  • Use Docker named volumes (no bind mounts for persistent data)
  • Store volumes in /home/login/data on host
  • Establish docker-network connection between containers
  • Configure domain: login.42.fr pointing to local IP
  • Use environment variables via .env file
  • Use Docker secrets for sensitive information
  • No passwords in Dockerfiles

Critical Restrictions

  • ❌ No ready-made Docker images from DockerHub
  • ❌ No network: host or --link
  • ❌ No infinite loops: tail -f, sleep infinity, while true
  • ❌ No latest tag
  • ❌ No passwords in Git repository
  • ✅ Containers must restart on crash
  • ✅ NGINX is the only entrypoint (port 443)

Infrastructure Design

Data Persistence Strategy

Two Docker named volumes provide data persistence:

  • WordPress Database Volume: Stores MariaDB data
  • WordPress Files Volume: Stores website files and uploads

Both volumes mount to /home/login/data on the host machine, ensuring data survives container recreation.

Network Architecture

All containers communicate through a dedicated Docker network. NGINX acts as a reverse proxy, receiving external HTTPS requests and forwarding them to WordPress. WordPress connects to MariaDB for database operations.

Internet (HTTPS:443)
    ↓
[NGINX Container] ← TLS Termination
    ↓
[WordPress + php-fpm Container]
    ↓
[MariaDB Container]
    ↓
Docker Volumes ← Persistent Storage

Security Considerations

  • TLS 1.2/1.3 encryption for all external traffic
  • Secrets stored outside Git repository
  • Environment variables for configuration
  • No root passwords in plain text
  • Isolated container networking
